Журнал заданий: автонажатие неактивных заданий при скролле

Проблема

При прокрутке журнала заданий система автоматически «нажимает» неактивные задания. Нет задержки между жестом скролла и действием выбора. Кладовщик видит задания, которые ещё не объявлены — это создаёт путаницу и лишние переходы.

Предлагаемые решения
  • Тайминг Ввести debounce / delay 100–500 мс между окончанием скролла (или touch-end) и обработкой tap/click. Пока идёт scroll — блокировать click / pointerup на элементах списка.
  • Фильтр В журнале показывать только активные задания, объявленные кладовщику. Остальные — не рендерить (аналог display: none / условный рендер на сервере).
  • Избранное Часто используемые задания — в localStorage (офлайн-приоритет) и/или в ЛК / БД профиля кладовщика; сортировать список: избранные → активные → остальные.
Варианты реализации тайминга:
  • A: 300 мс после scrollend / touchend без движения.
  • B: Разделить зоны: скролл только по контейнеру, выбор — по отдельной кнопке «Открыть».
  • C: Long-press 400 мс для входа в задание (скролл не триггерит).
2

Поиск ГЕ: исчезает журнал заданий (в отличие от «Остаток товара»)

Проблема

Модуль «Остаток товара» при поиске работает стабильно — UI не вылетает, данные не пропадают. При поиске информации по ГЕ пропадают журнал заданий и выбор нескольких активных заданий. Кладовщику приходится выходить из учётной записи и входить заново.

Предлагаемые решения
  • Паттерн Унифицировать навигацию с модулем «Остаток товара»: не размонтировать корневой shell с журналом заданий; открывать поиск ГЕ как overlay / drawer или дочерний экран с сохранением state.
  • Кнопка Добавить в AWM постоянную кнопку «Журнал заданий» в шапке (как быстрый возврат) — видна на экранах поиска ГЕ.
  • Сессия Кэшировать выбранное задание в sessionStorage до явного выхода.
Варианты:
  • A: Split-view: журнал слева (свёрнут), контент справа.
  • B: Floating FAB «Задания» поверх экрана ГЕ.
  • C: Back stack без destroy предыдущего Activity/Fragment.
3

Перемещение МХ → МО: «вылетает» активный адрес

Проблема

Перемещение МО → МО работает без сбоев. При МХ → МО сбрасывается активный адрес — нужно снова идти на адрес. Много лишних переходов.

Предлагаемые решения
  • State Сохранять активный адрес / МХ в state машины перемещения до завершения операции или явного «Сменить адрес».
4

Клавиатура Google при сканировании ШК

Проблема

При сканировании ГЕ, адресов и др. всплывает клавиатура Google. Часто после ошибки (например, в «Остаток товара» отсканировали ГЕ вместо товара или ничего не отсканировали) клавиатура появляется на каждом следующем скане. Вероятная причина — поле input с фокусом и типом, провоцирующим IME на Android.

Предлагаемые решения
  • Код Поле по умолчанию без фокуса; inputmode="none", readonly до явного режима ручного ввода; скрытый буфер для сканера; showSoftInputOnFocus=false (Android).
  • Сброс После ошибочного скана — принудительно InputMethodManager.hideSoftInputFromWindow.
  • Обход-Костыль Приложение No Keyboard: Hideable keyboard или отключение Gboard в настройках ТСД.
Варианты (приоритет):
  • 1 (лучший): Исправление в AWM — не показывать IME при скане.
  • 2: Enterprise policy / настройки ТСД.
  • 3: Стороннее приложение-заглушка клавиатуры.
5

Приёмка: нет признака, что товар уже принят на ГЕ

Проблема

При сканирования и сохранения товара в ГЕ, снова попадаем в «Приёмку» этого же ГЕ без визуальных маркеров. Частая ошибка повторная приёмка того же товара на ту же ГЕ или нажатие «ГЕ ПРИНЯТА» без фактической приёмки 0.

Предлагаемое решение (лёгкое)
  • UX Кнопка «ГЕ ПРИНЯТА» по умолчанию серая (доступна, в т.ч. для пустой ГЕ).
  • После фактического сохранения товара на ГЕ — кнопка становится зелёной → кладовщик видит подтверждение и осознанно закрывает ГЕ.
  • Дополнительно: колличеств сохраненных товаров в данное ГЕ.
6

Мульти-операции с ГЕ: закрытие · привязка к месту · привязка товара

Общая идея

Три похожие проблемы — кладовщик повторяет одно действие десятки раз. Решение: режим «Мульти» — диапазон ГЕ (от / до), превью, одна кнопка. Подробнее и макеты →

6A · Закрытие пустых ГЕ

30 паллет принято, ~50 лишних пустых ГЕ. Решение: диапазон №37431…№37480 или чекбоксы → «Закрыть выбранные» (только пустые).

6B · Привязка к 0-0-1 (Разгрузка)

~70 ГЕ по одной. Решение: №37400…№37470 → привязка к 0-0-1, превью списка, импорт из задания.

6C · Товар на множество ГЕ

Если на каждую ГЕ вручную привязать по 10 стульев, уйдёт ~23 минуты. В мульти-режиме: выбираете один товар, сколько штук на ГЕ и номера ГЕ от и до — система делает всё за раз.
